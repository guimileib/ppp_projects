# Princípios e Padrões de Projetos 

Autor: Guilherme Mileib 

## Laboratório 1. OCP, SRP, ISP
3 exercícios de aplicação dos princípios OCP, SRP e ISP. Cada exercício contém uma descrição do problema, a solução proposta e o código-fonte correspondente.

## Laboratório 2. Sigleton and Factory 
Soluções das duas questões do laboratório. Em cada questão são usados os
padrões Factory (para construir os produtos a partir do nome do modelo)
e Singleton (para garantir uma única instância de cada fabricante).

## Laboratório 3. Adapter and Proxy
Implementando os padrões de projeto com o objetivo de adaptar a interface de uma classe para outra e controlar o acesso a um objeto.

## Laboratório 4. Decorator and Observer
Implementando os padrões de projeto com o objetivo de adicionar responsabilidades a objetos de forma dinâmica e notificar objetos sobre mudanças de estado em outros objetos.

## Laboratório 5. Strategy and Command
Implementando os padrões de projeto com o objetivo de encapsular algoritmos e permitir a troca de comportamento em tempo de execução, além de encapsular uma solicitação como um objeto, permitindo parametrizar clientes com diferentes solicitações.

## Laboratório 6. State and Template Method
Implementando os padrões de projeto com o objetivo de permitir que um objeto altere seu comportamento quando seu estado interno muda, além de definir o esqueleto de um algoritmo em uma operação, deixando alguns passos para subclasses implementarem.
